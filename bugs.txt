- exit should be a builtin. It should print "exit" when exiting. I'm not sure if we have to implement it, but in bash, if you write "exit 77", 77 will be the exit code of the program (parent process). exit doesn't have to work in pipes (it doesn't work in bash, but in zsh).

- ft_execvp is an implementation of execvp. Instead of changing the logic in the function, it's better to make a separate function that calls it. Now the function does a lot of different things, which makes it hard to trouble shoot.

- run_cmd is part of the main loop, it gets called by handle_child_process, handle_parent_process, handle_redir_cmd. It's part of the recursive logic. It never returns and only exits on error or when a command is finished executing. Forking is done in main, handle_child_process and handle_parent_process. Creating more forks in run_cmd might cause undefined behaviour and probably some zombie kids.
